# Product Requirements Document (PRD): DataBuddy AI

## 1. Executive Summary
DataBuddy AI is a Conversational Database Studio (IDE). It allows users to interact with a managed PostgreSQL database (Neon) using natural language. The system translates prompts into executable SQL, runs the commands safely, and visualizes the database schema and tables in real-time within a 3-panel workspace.

## 2. Core Functional Requirements

* **Requirement 1: AI Chatbot (The Command Center)**
  A persistent chat interface where users input natural language requests (e.g., "Create an Employees table"). It handles intent parsing and displays conversational feedback.
* **Requirement 2: SQL Execution & Live Progress (The Engine)**
  A dedicated panel that displays the exact SQL generated by the LLM. It includes a terminal-style logger showing real-time execution progress (e.g., `Connecting... -> Executing -> Success: 0 rows affected`).
* **Requirement 3: Real-Time DB Visualization (The Source of Truth)**
  A dynamic data grid and schema browser that automatically refreshes the moment a write/modify command is successfully executed in the SQL panel.

## 3. Primary User Journey (The "Create Table" Flow)
1. **Trigger:** User enters prompt: "Create a new table called Employees with Emp name, emp phone number, experience, salary."
2. **Translation:** The AI Chatbot displays a "Generating..." state.
3. **Code Reveal:** The SQL Panel types out the generated `CREATE TABLE` syntax.
4. **Execution:** The SQL Panel's terminal logs the connection to the Neon DB and confirms successful execution.
5. **Visualization:** The Real-Time DB Preview panel automatically refreshes, displaying the new, empty "Employees" table with the correct column headers.
