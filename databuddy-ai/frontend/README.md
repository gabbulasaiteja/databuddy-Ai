# DataBuddy AI Frontend

Single-page application built with Next.js 15, TypeScript, Tailwind CSS, and shadcn/ui components. Fully integrated with the FastAPI backend for real-time database operations.

## Project Structure

```
frontend/
├── app/
│   ├── layout.tsx          # Root layout with TooltipProvider, Toaster, and dark mode
│   ├── page.tsx             # Main page with 3-panel resizable layout
│   └── globals.css          # Tactile Dark theme styles
├── components/
│   ├── ui/                  # shadcn/ui components (23 components)
│   ├── Header.tsx           # Top header with app title and connection status
│   ├── SQLPanel.tsx         # Left panel: SQL code editor + terminal logs
│   ├── DBPreview.tsx        # Top right: Database table preview with selector
│   └── Chatbot.tsx          # Bottom right: Chat interface with message history
└── lib/
    ├── api.ts               # API client for backend endpoints
    ├── store.ts             # Zustand store for global state management
    └── utils.ts             # Utility functions (cn helper)
```

## Design System: "Tactile Dark"

- **Background:** `#0A0A0A` (True Black)
- **Panel Surface:** `#171717` (Dark Gray)
- **Borders:** `#262626` (Thin, tactical borders)
- **Text:** `#EDEDED` (Off-white)
- **Success/Live Status:** `#00E599` (Neon Green)
- **Typography:** Geist Sans (UI), Geist Mono (Code/Data)

## Layout

The application uses a 3-panel resizable workspace:

1. **Left Panel (50% width):** SQL Panel
   - Top: SQL code editor with syntax display (real-time from Grok AI)
   - Bottom: Terminal/execution log with real-time progress from backend

2. **Top Right Panel (50% height):** Real-Time DB Preview
   - Table selector dropdown (populated from database schema)
   - Data table with columns and rows (real data from PostgreSQL)
   - Refresh button to reload schema

3. **Bottom Right Panel (50% height):** Chatbot
   - Scrollable message history
   - Input console for natural language queries (uses Grok AI)

## Backend Integration

The frontend is fully integrated with the FastAPI backend:

### API Endpoints Used:
- `GET /api/schema` - Fetch database schema and tables
- `POST /api/translate` - Translate natural language to SQL using Grok AI
- `POST /api/execute` - Execute SQL queries and get results
- `GET /history` - Get query history (optional)

### State Management:
- Uses Zustand for global state
- Manages: schema, SQL queries, execution logs, chat messages, table previews
- Auto-refreshes schema after CREATE/ALTER/DROP operations

### Features:
- ✅ Real-time SQL generation from natural language (Grok AI)
- ✅ Real-time execution logs from backend
- ✅ Real-time database preview with actual data
- ✅ Auto-execution of generated SQL queries
- ✅ Error handling with toast notifications
- ✅ Loading states and skeleton screens
- ✅ Rate limit handling

## Setup

1. **Install dependencies:**
   ```bash
   cd frontend
   npm install
   ```

2. **Configure environment:**
   ```bash
   cp .env.local.example .env.local
   ```
   
   Edit `.env.local` and set:
   ```env
   NEXT_PUBLIC_API_URL=http://localhost:8000
   ```

3. **Start the backend:**
   ```bash
   cd ../backend
   # Make sure you have .env with DATABASE_URL, GROQ_API_KEY, etc.
   python -m uvicorn main:app --reload
   ```

4. **Start the frontend:**
   ```bash
   cd frontend
   npm run dev
   ```

The application will be available at `http://localhost:3000`

## Usage Flow

1. **Send a prompt** in the Chatbot panel (e.g., "Create a table called Employees with name, email, salary")
2. **Watch the SQL** appear in the SQL Panel (generated by Grok AI)
3. **See execution logs** in the Terminal section (real-time from backend)
4. **View the result** in the DB Preview panel (auto-refreshes after schema changes)

## Error Handling

- Rate limit errors show toast notifications
- SQL validation errors are displayed in chat and terminal
- Network errors are caught and displayed
- All errors are logged in the execution terminal
